# Source-To-Score Ledger

The core platform model is:

Entity -> Source Document -> Source Citation -> Metric Definition -> Metric Value -> Confidence Grade -> Validation -> Score Run -> Score -> Insight -> Report / Dashboard / Advisory Action

Every metric needs a source because sustainability data must be inspectable and defensible. A number without a citation is not intelligence; it is only an assertion.

Every score needs a methodology version because scoring rules will evolve. The methodology version preserves reproducibility, auditability, and dispute handling.

Confidence grades are first-class because all source evidence is not equal. Structured public data, official PDFs, proxy estimates, voluntary disclosures, and missing values must influence interpretation differently.

Manual review comes before automation because early sustainability intelligence quality depends on review discipline. Automation can accelerate extraction later, but review status and citation quality must remain visible.

This ledger supports:

- Intelligence: source-backed metrics and scores.
- Advisory: defensible recommendations and evidence trails.
- Connect: trusted data exchange between stakeholders.
- Impact: measurable interventions and longitudinal outcomes.
- Academy: explainable methodology, literacy, and public research.
